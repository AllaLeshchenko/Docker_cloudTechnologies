1. Что такое Docker? - 
 b) Контейнерная платформа для упаковки, доставки и запуска приложений.
Docker позволяет разработчикам упаковать приложения и все их зависимости в единый контейнер. Эти контейнеры можно запускать на любом сервере, где установлен Docker, что упрощает развертывание и масштабирование приложений.


2.Соотнесите описание(поданы ниже) и понятие (Docker, Классическая виртуализация):
- Каждая виртуальная машина имеет свою собственную операционную систему => Классическая виртуализация;
- Все контейнеры на одном хосте используют общую операционную систему => Docker;
- Запускает полные копии операционной системы => Docker;
- Запускает полные копии операционной системы => Классическая виртуализация;
- Использует контейнеризацию для упаковки и запуска приложения => Docker;
- Использует гипервизор для виртуализации аппаратных ресурсов => Классическая виртуализация.


3.  Верно ли утверждение:
Все контейнеры, запущенные на одной машине, делят ядро операционной системы, поэтому они стартуют немедленно и более эффективно.
а) Да
Все контейнеры, запущенные на одном хосте, делят ядро ОС. Это делает их легковесными и позволяет запускать их почти мгновенно. Это ключевое отличие от виртуальных машин, где у каждой VM своё ядро.


4. Какой из компонентов Docker отвечает за выполнение и управление контейнерами?
a) Docker Engine
Docker Engine — это серверный компонент, который запускает и управляет контейнерами. Он взаимодействует с другими инструментами Docker (CLI, Compose и др.) и обеспечивает работу контейнеров.


5. К какому из компонентов Docker относится описание:
Изолированные среды, которые запускаются на основе образов и могут содержать всё необходимое для выполнения приложения.
c) Docker Container
Контейнер — это изолированная среда, созданная на основе Docker Image. Внутри контейнера — приложение, библиотеки и зависимости. Он работает быстро и изолированно от других контейнеров


6. Что представляет собой Dockerfile?
a) Текстовый файл, описывающий шаги по созданию Docker образа.
Dockerfile содержит инструкции: какую ОС использовать, что установить, какие файлы скопировать, какие команды выполнить. По нему создаётся образ (image), из которого потом запускается контейнер.


7.  Что представляют собой Docker Images?
b) Шаблоны для создания контейнеров, состоящие из слоев
Image (образ) — это шаблон, содержащий всё, что нужно для запуска приложения: код, зависимости, настройки. Он состоит из слоёв (каждая инструкция Dockerfile — отдельный слой). Из образа можно запустить сколько угодно контейнеров.


8. Что такое Docker Hub и другие Docker реестры (registries)?
b) Онлайн-платформа для обмена, хранения и управления Docker образами.
Docker Hub — это самый известный публичный реестр (registry), где разработчики хранят и делятся своими образами. Есть также приватные реестры (например, Harbor или GitHub Container Registry).
